# Versioning and Changelog Policy

## Policy

- Use semantic versioning for public-facing artifacts where applicable.
- Ship changelog notes with each release-affecting pull request.
- Keep release notes focused on operator impact:
  - Added
  - Changed
  - Fixed
  - Deprecated/Removed

## Source of truth

- GitHub releases and merged PR history in canonical repos.
- Operational rollout notes in `docs/ROLL_OUT.md`.

## Minimum release note bar

- What changed
- Why it changed
- Any operator action required
- Any rollback note if behavior changed
